# Reading the Output

## `report` summary

```
─── Claude Stats — week (America/New_York) ───

Sessions : 12
Prompts  : 94
Input    : 1.3M
Output   : 287K
Cache    : 620K read, 98K created (32.3% efficiency)
Models   : claude-opus-4-6 (10), claude-sonnet-4-6 (2)
Top tools: Read:201  Edit:88  Bash:44  Glob:19  Grep:17
```

### Fields

| Field | Meaning |
|---|---|
| **Sessions** | Number of Claude Code sessions in the selected period |
| **Prompts** | Number of user messages sent (each time you pressed Enter or submitted a prompt) |
| **Input** | Total input tokens billed across all assistant responses |
| **Output** | Total output tokens generated by the model |
| **Cache** | Cache read tokens (retrieved from Claude's prompt cache) and cache creation tokens (written into the cache); see below |
| **Models** | Models used, with the count of sessions using each |
| **Top tools** | Most-used built-in tools across all sessions, sorted by call count |

### Token counts

Token counts are sourced directly from the `usage` object in each assistant response. They match what Claude's API reports and what your plan is billed for.

- **Input tokens** — prompt content sent to the model on each turn.
- **Output tokens** — content generated by the model.
- **Cache read tokens** — tokens retrieved from the prompt cache (much cheaper than input tokens).
- **Cache creation tokens** — tokens written into the cache (slightly more expensive than input).

### Cache efficiency

```
Cache efficiency = cache_read / (input + cache_read) × 100
```

A higher percentage means more of the context was served from cache, reducing cost and latency. For long sessions with large system prompts, 30–60% efficiency is typical.

### Token formatting

| Displayed | Actual range |
|---|---|
| `500` | 0–999 |
| `12K` | 1 000–999 999 |
| `1.4M` | ≥ 1 000 000 |

## `report` — By Repo table

When sessions span more than one git remote origin, a per-repo breakdown is shown (before the per-project breakdown). This is useful when you have cloned the same repo into multiple local directories:

```
─── By Repo ───

                                                    Sess  Prompts     Input    Output
──────────────────────────────────────────────────────────────────────────────────────
https://github.com/org/myrepo                         12       94      1.3M     287K
https://github.com/org/other                           4       32      320K      89K
(no remote)                                            2       10       80K      22K
```

Sessions in directories that are not git repos, or whose repos have no `origin` remote, appear under `(no remote)`.

To filter the report to a single repo regardless of which local clone was used:

```sh
claude-stats report --repo https://github.com/org/myrepo
```

## `report` — By Project table

When sessions from more than one project path are present, a per-project breakdown is appended:

```
─── By Project ───

Project                                    Sess  Prompts     Input    Output
────────────────────────────────────────────────────────────────────────────
/Users/you/repos/myproject                     8       62      980K     198K
/Users/you/repos/other                         4       32      320K      89K
```

Projects are sorted by total input tokens (highest first). The By Repo table is omitted when `--repo` is active; the By Project table is omitted when `--project` is active.

## `status` output

```
─── Claude Stats Status ───

Database size   : 1.2 MB
Sessions        : 42
Messages        : 1 876
Quarantined     : 0 unparseable lines
Last collected  : 3/8/2026, 9:15:04 AM
```

| Field | Meaning |
|---|---|
| **Database size** | Size of `~/.claude-stats/stats.db` on disk |
| **Sessions** | Total sessions stored (all time, all projects) |
| **Messages** | Total assistant messages stored (one per API call) |
| **Quarantined** | Lines that could not be parsed; see [faq.md](faq.md) |
| **Last collected** | Wall-clock time of the most recent `collect` run |

## Web dashboard (`serve` / `report --html`)

The dashboard presents the same data as the terminal report in graphical form. It is produced either by `claude-stats serve` (live, browser-based) or by `claude-stats report --html` (static file).

### Summary bar

At the top of the page, six stats are always visible:

| Stat | Source |
|---|---|
| **Sessions** | Number of sessions in the selected period |
| **Prompts** | Total user messages |
| **Tokens** | Input + output tokens combined (formatted as K or M) |
| **Cache** | Cache efficiency percentage |
| **Cost** | Estimated API-equivalent cost |

### Chart panels

| Panel | Chart type | What it shows |
|---|---|---|
| **Daily trend** | Line | Tokens per day (left axis) and estimated cost per day (right axis) |
| **Model split** | Doughnut | Share of tokens consumed by each Claude model |
| **Top projects** | Horizontal bar | Up to 10 projects ranked by total tokens; label is the last path segment |
| **Entrypoint** | Pie | CLI (`claude`) vs VS Code (`claude-vscode`) session counts |
| **Stop reasons** | Bar | Count of `end_turn`, `tool_use`, and `max_tokens` stops |
| **Cache** | Doughnut | Cache-read tokens vs uncached input tokens |

### Period selector

The dropdown at the top of the page switches between Day / Week / Month / All. Selecting a period reloads the page with a `?period=` query parameter, which the server uses to re-query the database for that time range.

In a static HTML export (`report --html`), the data is a fixed snapshot — the period selector is not present.

### Filtering by project or repo

The dashboard shows all projects by default. To narrow it to one project, add a `project` or `repo` query parameter to the URL:

```
http://localhost:9120/?project=/Users/you/repos/myproject
http://localhost:9120/?period=week&project=/Users/you/repos/myproject
```

See the [`serve` command reference](commands.md#serve) for the full list of supported URL parameters.

### Auto-refresh

The **Auto-refresh** button reloads the page every 30 seconds — useful for leaving the dashboard open during an active coding session. Auto-refresh is only available when running `claude-stats serve`; static HTML exports do not update after export.

## `export` JSON format

Each element in the JSON array is a session object:

```json
{
  "session_id": "abc123...",
  "project_path": "/Users/you/repos/myproject",
  "source_file": "/Users/you/.claude/projects/-Users-you-repos-myproject/abc123.jsonl",
  "first_timestamp": 1700000000000,
  "last_timestamp": 1700003600000,
  "claude_version": "2.1.70",
  "entrypoint": "claude",
  "git_branch": "main",
  "is_interactive": 1,
  "prompt_count": 8,
  "assistant_message_count": 8,
  "input_tokens": 12400,
  "output_tokens": 3100,
  "cache_creation_tokens": 800,
  "cache_read_tokens": 5200,
  "tool_use_counts": "[{\"name\":\"Read\",\"count\":12}]",
  "models": "[\"claude-opus-4-6\"]",
  "source_deleted": 0
}
```

Timestamps are Unix milliseconds (UTC). `tool_use_counts` and `models` are JSON-encoded strings within the outer JSON.
